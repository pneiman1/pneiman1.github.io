Deploy Hexo To GitHub Pages

Hi folks! In my last blog post, we covered installing the Hexo blog engine via Command Line Interface (CLI), running the Hexo server, creating a new blog post, and generating the static files needed to deploy your blog to production.

Today, we’re picking up right where we left off, and we’re going to deploy our Hexo blog to GitHub Pages, the publishing arm of the GitHub platform.

Configuration

So, above, check out the line: url: http://gitusername.github.io/. With this exact url in mind, head over to GitHub and replicate this by creating a new repository, replacing gitusername with your GitHub account’s username. Now you should have the remote repository created that will host your code online, and in your main _config.yml file your url has been changed from localhost:4000 to url: http://gitusername.github.io/. Beautiful.
